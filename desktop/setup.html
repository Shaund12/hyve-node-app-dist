Welcome to the Hyve Validator Dashboard
This guided wizard will walk you through every step of installing the Hyve Validator Dashboard on your machine. The dashboard is a powerful monitoring and management tool for your Hyve blockchain validator node — it provides real-time metrics, staking operations, reward tracking, governance voting, and much more.
Important: Prerequisites Required
Before continuing, you must have completed all of the following. If you haven't, please do so first and return to this installer afterward.
📋 1. Installed the Hyve Validator App
Download the Hyve Validator application for your platform (Windows or Linux) from the official Hyve website. This app installs the hyved binary and sets up the node directory structure at ~/.config/hyve-node/.
📋 2. Activated Your Validator
Used the app to create and activate your validator on the Hyve network. This involves self-delegating at least 1 HYVE and registering your validator on-chain. Your validator must have a valoper address (starts with hyvevaloper...).
📋 3. Node Has Synced
Your node should have completed an initial sync at least once. The node doesn't need to be running right now — but it must have been synced before. If you just installed the app and it's still syncing, wait for it to finish first. You can check sync status in the Hyve Validator app or by running:
curl -s http://127.0.0.1:26657/status | jq .result.sync_info.catching_up
(should return false when synced)
▶ Set up on Windows first? Click here for migration instructions

If you activated your validator using the Hyve Validator app on a Windows PC and want to move it to an Ubuntu server for 24/7 operation, follow these steps.

Step 1: Install the Hyve Validator App on Ubuntu

Download and install the Linux version of the Hyve Validator app on your Ubuntu machine. Run it once and let it initialize — this creates the directory structure and downloads the Linux hyved binary.

Default install location: ~/.config/hyve-node/

Important: Stop the validator app on Ubuntu after the initial setup. Do not let it sync yet — you need to copy your Windows config first.

Step 2: Locate Your Windows Config Files

On your Windows PC, the Hyve Validator app stores data at:

%APPDATA%\hyve-node\

Open File Explorer and paste that path in the address bar, or navigate to:

C:\Users\YOUR_USERNAME\AppData\Roaming\hyve-node\

Inside, you'll see a structure like:

hyve-node/ ├── bin/ ← Windows binaries (do NOT copy these) ├── home/ │ ├── config/ ← ✅ COPY THIS ENTIRE FOLDER │ │ ├── app.toml │ │ ├── config.toml │ │ ├── genesis.json │ │ ├── node_key.json │ │ └── priv_validator_key.json ← ⚠️ CRITICAL FILE │ ├── data/ ← blockchain data (do NOT copy — will re-sync) │ └── keyring-test/ ← ✅ COPY THIS if it exists └── logs/

Step 3: Copy Config Files to Ubuntu

You need to copy the config/ folder and optionally the keyring-test/ folder from your Windows machine to your Ubuntu machine. Here are several ways to do it:

Option A: USB Drive

  1. On Windows: Copy the home\config\ folder to a USB drive
  2. Also copy home\keyring-test\ if it exists
  3. Plug the USB into your Ubuntu machine
  4. Copy the files to the node directory:
# Replace /media/YOUR_USER/USB_NAME with your actual USB mount path cp -r /media/$USER/USB_NAME/config/* ~/.config/hyve-node/home/config/ cp -r /media/$USER/USB_NAME/keyring-test ~/.config/hyve-node/home/ 2>/dev/null

Option B: SCP (network transfer)

If both machines are on the same network, use SCP from Windows (PowerShell or Git Bash):

# Run this on your Windows PC (PowerShell) # Replace UBUNTU_USER and UBUNTU_IP with your Ubuntu username and IP address scp -r "$env:APPDATA\hyve-node\home\config\*" UBUNTU_USER@UBUNTU_IP:~/.config/hyve-node/home/config/ scp -r "$env:APPDATA\hyve-node\home\keyring-test" UBUNTU_USER@UBUNTU_IP:~/.config/hyve-node/home/

Or from your Ubuntu machine, pull from Windows (if Windows has OpenSSH enabled):

# Run this on your Ubuntu machine # Replace WIN_USER and WIN_IP with your Windows username and IP address scp -r WIN_USER@WIN_IP:"C:/Users/WIN_USER/AppData/Roaming/hyve-node/home/config/*" ~/.config/hyve-node/home/config/ scp -r WIN_USER@WIN_IP:"C:/Users/WIN_USER/AppData/Roaming/hyve-node/home/keyring-test" ~/.config/hyve-node/home/

Option C: WinSCP (GUI transfer)

  1. Download WinSCP on your Windows PC
  2. Connect to your Ubuntu machine using its IP address and your SSH credentials
  3. Navigate to ~/.config/hyve-node/home/ on the Ubuntu side
  4. Drag and drop the config/ folder from Windows to Ubuntu, overwriting existing files
  5. Also drag keyring-test/ if it exists

Step 4: Set Correct Permissions

After copying, fix file permissions on Ubuntu:

chmod 600 ~/.config/hyve-node/home/config/priv_validator_key.json chmod 600 ~/.config/hyve-node/home/config/node_key.json chmod -R 700 ~/.config/hyve-node/home/keyring-test 2>/dev/null

Step 5: Export Your Private Key

Before shutting down the Windows validator, export your private key. You'll need it for the dashboard's Configuration step (Step 5 in this wizard):

  • From Hyve Validator App: Settings → Export Private Key
  • From MetaMask: Account Details → Export Private Key

Save the 64-character hex key somewhere safe. You'll paste it in Step 5.

Step 6: Stop the Windows Validator

CRITICAL — Double-Sign Prevention
Before starting the node on Ubuntu, you must fully stop the validator on Windows. Running the same validator on two machines simultaneously will cause a double-sign, which results in permanent jailing and slashing of your staked HYVE.
  1. Close the Hyve Validator app on Windows completely (check the system tray)
  2. Verify no hyved process is still running: open Task Manager and check
  3. Optionally uninstall the Hyve Validator app to prevent accidental restart

Step 7: Continue with This Wizard

Once you've completed the steps above:

  • Your Ubuntu machine has the Linux hyved binary (from the Linux Hyve Validator app)
  • Your validator's config and keys are copied from Windows
  • The Windows validator is fully stopped
  • You have your private key saved for the Configuration step

Click "Get Started" below to continue with the dashboard setup.

What This Installer Does
  1. Checks your system for required software (Python, curl, etc.)
  2. Locates your Hyve node installation and verifies it's set up correctly
  3. Creates a Python virtual environment and installs all dashboard dependencies
  4. Configures your validator private key (optional — dashboard works in read-only mode without it)
  5. Sets up PostgreSQL database for metrics storage (optional — falls back to JSON files)
  6. Installs and starts two systemd services that run automatically on boot
Installation directory: ~/.local/share/hyve-dashboard/
Dashboard URL: http://127.0.0.1:8420 (local only by default)
Time to complete: About 2–5 minutes depending on your internet speed
System Requirements
The dashboard needs a few system packages to run. This step automatically detects what's installed on your system. Green checkmarks mean you're good — red X marks mean something needs to be installed before continuing.
Required Packages
  • Python 3.10+ — Runs the FastAPI backend server that powers the dashboard
  • python3-venv — Creates an isolated virtual environment so dashboard packages don't conflict with your system Python
  • curl — Used by the dashboard to query your node's RPC endpoints
Optional: PostgreSQL — Enables persistent storage for metrics history, reward logs, delegation tracking, and more. Without it, the dashboard stores data in local JSON files which works fine but limits history to a smaller window.

Locate Your Hyve Node
The dashboard connects to your local Hyve node to read validator status, monitor performance, and execute transactions. We need to verify that your node installation is set up correctly.
What we're looking for:
The Hyve Validator app installs the node to ~/.config/hyve-node/ by default. This directory should contain:
  • bin/hyved — The validator binary that runs your node
  • home/config/ — Node configuration files (app.toml, config.toml, genesis.json)
  • home/data/ — Blockchain state database
  • home/keyring-test/ — Key storage used for transaction signing
If you installed the Hyve Validator app to a custom location, update the path below. The path should point to the root directory (the one containing bin/ and home/).
The directory containing bin/hyved, home/config/, and home/data/. The default path works for most installations.
Troubleshooting
  • Directory not found: Make sure you've installed and run the Hyve Validator app at least once
  • Binary missing: The app may still be downloading — open it and wait for the download to complete
  • Config missing: Run the Hyve Validator app and let it initialize (it creates config on first run)
  • Node not running: That's OK! The node doesn't need to be running during setup. The systemd service will start it automatically
Install Dependencies
This step creates an isolated Python virtual environment and installs all the packages the dashboard needs to run. Everything is installed locally — it won't affect your system Python installation.
What gets installed:
  • FastAPI + Uvicorn — The high-performance web server that serves the dashboard
  • httpx — HTTP client for communicating with your node's RPC/REST/EVM endpoints
  • psutil — System monitoring (CPU, memory, disk usage, process detection)
  • websockets — Real-time live updates pushed to the dashboard UI
  • asyncssh — SSH client for the failover feature (connecting to backup nodes)
  • asyncpg (optional) — PostgreSQL driver for persistent metrics storage
  • eth-account (optional) — EVM transaction signing for SHADE token operations
  • rlp (optional) — RLP decoder for parsing EVM transaction data
Optional packages are installed with best-effort — if they fail, the dashboard simply disables those specific features and everything else works normally.
This downloads and installs packages from PyPI. Typically takes 30–90 seconds depending on internet speed. Watch the terminal output below for progress.

Full mode requires your validator's private key and enables transaction signing — you'll be able to claim rewards, delegate, compound, vote on governance proposals, and manage SHADE tokens directly from the dashboard. Read-only mode works without a key — you get full monitoring but no transaction capabilities.

Install & Start Services
The dashboard runs as two systemd services that start automatically when your machine boots. This is the recommended way to run both the node and the dashboard — it ensures they survive reboots and restart automatically if they crash.
⚙️ hyve-node.service
Manages the hyved validator process. Sets up the correct environment (LD_LIBRARY_PATH, working directory) and redirects node logs to ~/.config/hyve-node/logs/app.log. Restarts automatically on failure with a 10-second delay.
📊 hyve-dashboard.service
Runs the Python web dashboard on port 8420 using Uvicorn. Depends on hyve-node.service — the dashboard won't start unless the node service is active. Serves the web UI, REST API, WebSocket feeds, and handles all blockchain interactions. Restarts automatically on failure with a 5-second delay.
What happens when you click Install:
  1. Your .env configuration file is saved with the settings from the previous step
  2. Service unit files are generated with your username and node path substituted in
  3. Services are copied to /etc/systemd/system/ (requires sudo)
  4. Both services are enabled to start on boot
  5. The node service is started (if not already running)
  6. The dashboard service is started
  7. If PostgreSQL is available, the database and user are created automatically
Note: A password dialog will appear asking for your system password. This is required to install systemd service files to /etc/systemd/system/ and manage services. You only need to authenticate once.

Manual Service Management
After installation, you can manage services with these commands:
sudo systemctl status hyve-dashboard — Check if dashboard is running
sudo systemctl restart hyve-dashboard — Restart the dashboard
sudo systemctl stop hyve-dashboard — Stop the dashboard
sudo journalctl -u hyve-dashboard -f — View live dashboard logs
sudo systemctl status hyve-node — Check if node is running
sudo journalctl -u hyve-node -f — View live node logs

Finding your password:

Dashboard Features
Here's what's available in your new dashboard:
  • Overview Tab — Live validator status, signed blocks, voting power, uptime percentage, CPU/memory/disk metrics with historical charts
  • Staking Tab — Delegate, redelegate, undelegate HYVE. Configure auto-compound. View delegation breakdown and staking history
  • Signing Tab — Track block signing performance, missed blocks, current signing window, jailing threshold monitor
  • SHADE Tab — View SHADE token balance, claim from emission contract, approve/transfer SHADE tokens
  • Governance Tab — View active and past proposals, vote Yes/No/Abstain/Veto, track community voting trends
  • Network Tab — Connected peers, peer geolocation map, network health, validator set changes
  • Logs Tab — Live streaming node logs in the browser, searchable and filterable
  • Settings Tab — Change password, configure Discord webhook alerts, manage alert conditions, maintenance notes/journal
  • Failover Tab — Configure a backup validator node, monitor its status, trigger failover if primary goes down
  • Upgrades Tab — Track chain upgrade proposals, download new binaries, manage upgrade lifecycle
